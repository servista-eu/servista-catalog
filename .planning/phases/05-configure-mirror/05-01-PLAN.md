    - `last_error` is empty string `""`

    Step 2: Trigger a manual mirror sync to ensure content is pushed NOW (not waiting for next interval):
    ```
    MIRROR_ID=$(curl -s -H "Authorization: token $FORGEJO_TOKEN" \
      "https://git.hestia-ng.eu/api/v1/repos/qabatz/qabatz-catalog/push_mirrors" | \
      python3 -c "import sys,json; mirrors=json.load(sys.stdin); print(mirrors[0]['remote_name'])" 2>/dev/null)

    curl -s -X POST -H "Authorization: token $FORGEJO_TOKEN" \
      "https://git.hestia-ng.eu/api/v1/repos/qabatz/qabatz-catalog/push_mirrors-sync/$MIRROR_ID"
    ```

    If the sync endpoint is not available (Forgejo version may vary), proceed to Task 2 verification -- the mirror should have already synced via sync_on_commit from prior pushes.

    Step 3: Re-query the mirror status after sync attempt to confirm `last_error` is still empty.
