**Step 2 (D-09): Forgejo API repo rename (per D-01 primary path)**

```bash
curl -s -X PATCH \
  -H "Authorization: token $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "qabatz-catalog"}' \
  "https://git.hestia-ng.eu/api/v1/repos/qabatz/qabatz-gradle-plugins"
```

If this returns HTTP 200, the rename succeeded. If it returns 403/404/422, execute the fallback migration path from D-01: create new repo `qabatz-catalog`, push all branches/tags, copy settings, then delete the old repo after confirming migration.

After the Forgejo rename, update the local remote to point to the new repo name:
```bash

```
